What is LIFO Page Replacement?

LIFO (Last In, First Out) is a page replacement algorithm used in operating systems to manage memory. When a page fault occurs and memory is full, the algorithm removes the most recently added page to make room for the new one.

Think of it like a stack of plates:

The last plate you put on top is the first one you take off. Similarly, the last page loaded into memory is the first one to be replaced.

How LIFO Works (Step-by-Step)

1

Check if page exists in memory

When a page is requested, first check if it's already in one of the available frames.

2

Page Hit (Found)

If the page is already in memory, it's a HIT. No replacement needed.

3

Page Fault (Not Found)

If the page is not in memory, it's a FAULT. The page must be loaded.

4

Check for empty frames

If there's an empty frame available, place the new page there and push it onto the stack.

5

Replace using LIFO

If all frames are full, remove the page at the top of the stack (most recently added) and replace it with the new page.

Comparison: LIFO vs FIFO vs LRU

Aspect LIFO FIFO LRU
Full Name Last In, First Out First In, First Out Least Recently Used
Replacement Rule Remove most recently added Remove oldest page Remove least recently accessed
Data Structure Stack Queue Priority Queue / Timestamps
Implementation Simple Simple Complex
Performance Generally poor Moderate Good (near optimal)
Use Case Educational / Rare Simple systems Modern OS & caches

Quick Facts

  • Uses a Stack data structure
  • O(1) replacement decision
  • Simple to implement
  • No access tracking needed

Pros

  • Very simple algorithm
  • Easy to implement
  • Low overhead
  • Constant time O(1)
  • Good for learning concepts

Cons

  • Poor hit ratio
  • Ignores page access patterns
  • Not used in real systems
  • Can cause thrashing
  • Belady's anomaly possible
